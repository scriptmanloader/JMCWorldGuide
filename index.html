Документация по сборке .jmcworld

Java Minecraft World

Формат .jmcworld (аббревиатура от Java Minecraft World) — это специализированный контейнер на базе ZIP-архива, созданный для упрощенного распространения контента. Он позволяет объединить мир, моды и настройки в один файл для автоматической установки.

Основная цель формата — избавить игрока от ручного копирования файлов по папкам saves, mods и resourcepacks.

1. Структура папок перед сборкой

Для успешной сборки создайте рабочую директорию со следующим содержимым:

MyEpicProject/
install.json (Должен быть здесь, в корне!)
WorldData/ (Файлы самого мира)
Addons/ (Моды или шейдеры)

Важно: Если файл install.json будет находиться внутри вложенных папок, сборщик выдаст ошибку.

2. Конфигурация install.json

Файл конфигурации определяет правила переноса данных в папку .minecraft.

Пример кода 
{
  "name": "Super Adventure Map",
  "version": "1.2.0",
  "actions": [
    {
      "source": "WorldData",
      "target": "saves/Super_Adventure"
    },
    {
      "source": "Addons/mods",
      "target": "mods"
    },
    {
      "source": "Addons/res",
      "target": "resourcepacks"
    }
  ]
}

3. Описание полей

Поле Описание
name Отображаемое имя проекта в списке установленных программ.
version Версия сборки (строка).
actions Массив объектов, каждый из которых описывает операцию копирования.
↳ source Путь к папке внутри архива.
↳ target Путь назначения внутри .minecraft (напр. "saves" или "mods").

4. Процесс сборки

1

Подготовка файлов

Убедитесь, что install.json находится строго в корневой папке вашего проекта (рядом с папками ресурсов).

2

Использование приложения

Вкладка "Разработчик (Сборка)" в программе JMCWorld Installer.

3

Генерация Java Minecraft World

Выберите папку с исходниками и укажите путь для сохранения. На выходе вы получите файл .jmcworld.

Механизм удаления

Установщик автоматически генерирует лог-файл *_uninstall.json в папке .minecraft/jmcworld_data. Это позволяет корректно удалять контент, не затрагивая файлы игрока.
